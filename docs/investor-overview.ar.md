# نظرة عامة للمستثمر (العربية)

## الملخص التنفيذي
DeliveryApp منصة متكاملة لتوصيل الطعام تستهدف المطاعم والعملاء ومندوبي التوصيل. تعتمد على .NET وABP، مع دعم المدفوعات (Stripe) والإعلانات والدردشة والإشعارات.

- المنتج: لوحة تحكم إدارية + واجهات ويب + API
- الجاهزية: قاعدة شيفرة كاملة مع نصوص تشغيل ونشر وتشخيص
- التفوّق: البناء على ABP لتسريع الهوية، الصلاحيات، وOpenIddict

## التقنية والمنتج
- .NET 9 + ABP + EF Core (SQL Server)
- المصادقة: OpenIddict (OIDC/OAuth2) + تفويض بالأدوار
- المدفوعات: Stripe (طلبات دفع، تحويلات للحسابات المتصلة، استرداد)، دعم الدفع عند التسليم
- الاتصالات: البريد عبر SendPulse، الإشعارات عبر Firebase

## نموذج العمل
- عمولة على الطلبات (قابلة للتهيئة لكل مطعم)
- رسوم منصة على المدفوعات والتحويلات
- إيرادات الإعلانات عبر `AdRequest` و`Advertisement`
- اشتراكات للخصائص المتقدمة (اختياري)

## مؤشرات الأداء المقترحة
- GMV، نسبة التحصيل (Take Rate)، صافي الإيرادات
- معدل نجاح الطلبات والدفع
- زمن التوصيل ونسبة الالتزام بالوقت
- CAC، الاحتفاظ D30/D90، MAU/DAU
- عائد الإنفاق الإعلاني ونسب التحويل

## خارطة الطريق
- تطبيقات جوال (React Native/Flutter) باستخدام الـ API الحالي
- توصيات وبحث متقدّم (ملفات التخصيص موجودة)
- كشف الاحتيال للمدفوعات وCOD
- دعم التعددية (Multi-tenancy) للمجموعات
- تحسين المراقبة والتتبّع (APM/لوحات)

## المخاطر والتخفيف
- الامتثال: الاستفادة من Stripe لـ KYC وتقليل نطاق PCI
- المنافسة: التركيز على الشراكات المحلية وأدوات المالك
- الموثوقية: CI/CD، بيئات زرقاء/خضراء، فحوصات صحّة
- الخصوصية: تدوير الأسرار، فصل البيانات الحساسة، تشفير السجلات

للتفاصيل التقنية والرسومات البيانية، راجع: `system-specification.ar.md`.
